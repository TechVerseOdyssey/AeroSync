# AeroSync 关键需求清单

> 按优先级排序的关键需求，反映当前实现状态

---

## 🚨 P0 级需求（阻塞性，必须立即做）

### 1. 身份认证系统 ✅ 已实现（核心部分）
**为什么是 P0**：当前任何人都可以上传文件，严重安全隐患

**最小可行方案**：
- [x] Token 认证（HMAC-SHA256 Bearer Token）
- [x] HTTP Header 携带 Token
- [x] 认证失败返回 401
- [x] 配置文件存储 Token（TOML）

**完整方案（v0.2）**：
- [ ] 用户名/密码认证
- [x] Token 过期机制
- [ ] IP 白名单
- [ ] 基于角色的权限（只读/读写/管理员）

**验收标准**：
- ✅ 未认证请求无法上传文件
- ✅ 错误的 Token 返回明确错误信息
- ✅ 配置文件示例完善

---

### 2. 传输加密 🟡 部分实现
**为什么是 P0**：HTTP 明文传输，敏感数据泄露风险

**最小可行方案**：
- [ ] HTTPS 支持（自签名证书）
- [ ] 提供证书生成脚本
- [ ] 文档说明如何配置

**完整方案（v0.2）**：
- [ ] TLS 1.3 强制加密（HTTP 端）
- [ ] Let's Encrypt 自动申请
- [x] QUIC 内置加密（rustls + TLS 1.3 已启用）
- [ ] 证书管理工具（GUI）

**验收标准**：
- ⚠️ HTTP 端口仍为明文，需补充 HTTPS 支持
- ✅ QUIC 传输已加密
- [ ] 证书过期提醒

---

### 3. 完整性校验 ✅ 已实现
**为什么是 P0**：无法保证传输后文件未损坏

**最小可行方案**：
- [x] 传输前计算 SHA-256
- [x] 传输后验证哈希
- [x] 不匹配时提示错误

**完整方案（v0.2）**：
- [ ] 大文件分块校验
- [ ] 校验失败自动重传
- [ ] 校验进度显示
- [ ] 支持 MD5/SHA-1/SHA-256 选择

**验收标准**：
- ✅ 手动损坏文件后传输失败
- ✅ 校验耗时 <1% 传输时间
- ✅ GUI 显示"校验通过 ✅"

---

### 4. 审计日志 ✅ 已实现（核心部分）
**为什么是 P0**：无法追溯谁上传了什么，合规风险

**最小可行方案**：
- [x] 记录上传/下载操作（时间、文件名、IP）
- [x] 写入文本日志文件
- [ ] 日志轮转（按天）

**完整方案（v0.2）**：
- [ ] JSON Lines 格式
- [ ] 包含：用户、时间、操作、文件、大小、IP、结果
- [ ] SQLite 数据库存储
- [ ] GUI 查询界面
- [ ] 导出功能

**验收标准**：
- ✅ 每次传输都有日志记录
- ✅ 日志可追溯至少 30 天
- ✅ 敏感信息已脱敏

---

## 🔴 P1 级需求（重要，显著影响体验）

### 5. 传输前预检查 ✅ 已实现（`preflight.rs`）
**问题**：传输到一半才发现磁盘满了，浪费时间

**实现**：
- [x] 检查目标磁盘剩余空间
- [x] 网络连通性测试（ping）
- [x] 预估传输时间
- [x] 显示"预检查通过 ✅"

**用户价值**：给用户"心理安全感"，提前发现问题

---

### 6. 错误信息友好化 ⏰ 估算：1 周
**问题**：`Error: Os { code: 13 }` 用户看不懂

**实现**：
- [ ] 定义 Top 10 常见错误
- [ ] 每个错误有人类可读说明
- [ ] 提供解决建议
- [ ] 相关文档链接

**示例**：
```
❌ 权限不足
无法写入目标文件夹：/var/uploads

可能原因：
1. 文件夹权限不足（当前需要 755）
2. 磁盘只读挂载
3. 当前用户不是文件夹所有者

建议解决方案：
• 使用 sudo 运行：sudo aerosync server
• 修改权限：chmod 755 /var/uploads
• 查看文档：https://aerosync.dev/docs/permissions

[📋 复制错误信息] [📁 查看日志]
```

---

### 7. 传输历史记录 ✅ 已实现（`history.rs`）
**问题**：传输失败后无法回溯，不知道哪些文件成功了

**实现**：
- [x] SQLite 数据库存储历史
- [x] 记录：时间、文件名、大小、速度、状态、错误
- [ ] GUI 历史面板
- [ ] 搜索和筛选功能
- [ ] 导出 CSV

**用户价值**：可追溯、可审计、可统计

---

### 8. 带宽控制 ✅ 已实现（`ratelimit.rs`）
**问题**：传输占满带宽，影响其他业务

**实现**：
- [x] 全局限速（MB/s）
- [x] 单任务限速
- [ ] 按时间段调度（夜间满速，白天限速）
- [ ] GUI 限速滑块

**用户价值**：避免影响生产业务

## 📊 优先级决策依据

### P0（阻塞性）标准
- ✅ 安全风险（数据泄露、未授权访问）
- ✅ 数据完整性风险（文件损坏无法检测）
- ✅ 合规要求（审计日志）
- ✅ 阻止生产使用

### P1（重要）标准
- ✅ 显著影响用户体验
- ✅ 降低支持成本（减少用户求助）
- ✅ 竞品标配功能
- ✅ 使用频率高

### P2（增强）标准
- ✅ 改善体验但非必需
- ✅ 特定场景需要
- ✅ 使用频率中等

### P3（长期）标准
- ✅ 差异化竞争力
- ✅ 战略性功能
- ✅ 开发成本高
