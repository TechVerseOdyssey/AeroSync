# v0.3.0 Frozen API Contract

> **Phase 0 deliverable** — see `docs/v0.3.0-refactor-plan.md` §3 Phase 0.  
> **Status**: Frozen — v0.3.0-rc1 (commit `b4a8e0c`) 公开符号集与本文档一致.  
>   Baseline taken from v0.2.1 (`f66eba7`); rc1 lands the planned splits without breaking any frozen surface listed below.  
> **Audience**: Every subagent / contributor working on v0.3.0 refactor phases 1–5.

## 0. How to use this document

The v0.3.0 refactor splits the codebase into `aerosync-domain` / `aerosync-infra` crates and introduces a `TransferSession` aggregate root. Refactor success is defined as:

> **Every public symbol listed below remains importable / callable / wire-compatible after v0.3.0 ships, with byte-identical Python / MCP / CLI / proto behaviour.**

Before any phase-N PR merges, the implementer MUST:

1. Run `rg <symbol>` against this doc; confirm the symbol still resolves.
2. Run the `Acceptance verification` script in §10.
3. If the refactor *requires* breaking a frozen symbol, **stop**: open a discussion to amend this document and bump v0.3.0 → v0.4.0 instead. No sneaky breakages.

Anything **not** listed here is fair game to refactor / rename / move freely.

## 2. Wire format (`aerosync-proto`) — fully frozen

`proto/aerosync/wire/v1.proto` is the **single source of truth** for the bytes on the wire under ALPN `aerosync/1`. **Every field number is permanent.** New fields may be added (additive); existing fields MUST NOT be renamed, renumbered, or have their type changed.

### 2.1 Messages (frozen)

| Message | Frozen fields |
|---|---|
| `Handshake` | 1=protocol_version, 2=sender_node_id, 3=auth_proof, 4=capabilities |
| `TransferStart` | 1=receipt_id, 2=file_name, 3=size_bytes, 4=sha256, 5=metadata, 6=chunk_size |
| `Metadata` (RFC-003) | 1=id, 2=from_node, 3=to_node, 4=created_at, 5=content_type, 6=size_bytes, 7=sha256, 8=file_name, 9=protocol; 20=trace_id, 21=conversation_id, 22=parent_file_ids, 23=expires_at, 24=lifecycle, 25=correlation_id; 99=user_metadata |
| `Cancel` | 1=receipt_id, 2=reason |
| `Heartbeat` | 1=monotonic_ms |
| `ControlFrame` (oneof) | 1=handshake, 2=transfer_start, 3=cancel, 4=heartbeat |
| `BytesReceived` | 1=bytes |
| `Received` | 1=checksum_ok, 2=sha256 |
| `Failed` | 1=code, 2=detail |
| `Acked` | reserved 1 (RFC-003 §4 ack-time metadata) |
| `Nacked` | 1=reason |
| `ReceiptFrame` (oneof body, 99=at) | 2=bytes_received, 3=received, 4=failed, 5=acked, 6=nacked |

### 2.2 Enums (frozen — never reuse, only extend at higher numbers)

| Enum | Variants |
|---|---|
| `Lifecycle` | 0=UNSPECIFIED, 1=TRANSIENT, 2=DURABLE, 3=EPHEMERAL |
| `ErrorCode` | 0=UNSPECIFIED, 1=AUTH, 2=CHECKSUM, 3=DISK_FULL, 4=PERMISSION, 5=TIMEOUT, 6=CANCELLED_REMOTE, 99=INTERNAL |
| `State` | 0=INITIATED, 1=STREAM_OPENED, 2=DATA_TRANSFERRED, 3=STREAM_CLOSED, 4=PROCESSING, 5=COMPLETED, 6=FAILED |

### 2.3 Field numbering reservation rules (per `v1.proto` lines 9-14)

```
1-19    : system fields, set by AeroSync engine
20-99   : well-known optional fields owned by SDK
≥100    : reserved for future system extensions
≥200    : reserved for SDK use
≥1000   : reserved forever for user-proto extensions
```

### 2.4 HTTP wire additions (Batch B + D.5, frozen)

| Header / body | Format | Direction |
|---|---|---|
| `X-Aerosync-Metadata` | `base64(protobuf(Metadata))` | sender → receiver, on `POST /upload[/{path}]` |
| 200 OK body `receipt_ack` field | `{ receipt_id: uuid, decision: "ack"\|"nack", reason: string\|null }` | receiver → sender, when inbound metadata carries parseable `id` |
| 400 body `receipt_ack` field | same shape, `decision: "nack"`, `reason: "sha256 mismatch"` | receiver → sender on SHA-256 mismatch |

When metadata absent or `id` unparseable → `receipt_ack` field omitted (back-compat with v0.2.0 senders).

### 2.5 QUIC stream multiplexing (Batch C, frozen)

| Stream type | First byte | Carries |
|---|---|---|
| Control | `0x00` (`CONTROL_STREAM_SENTINEL`) | length-delimited `ControlFrame` (TransferStart, Heartbeat, Cancel) + receiver-side `ReceiptFrame` (BytesReceived, Received, Acked, Nacked, Failed) |
| Data | ASCII (`UPLOAD:` prefix) | legacy v0.2.0 chunked file body |
